# Support

## Getting help

If you have questions about usage, architecture, or contribution:

1. Read the [README](./README.md) first.
2. Check if a similar issue already exists.
3. Open a new issue with enough technical context.

## What to include in support requests

- environment (browser, OS)
- expected vs actual behavior
- reproduction steps
- screenshots or short recording (when helpful)
- console logs/errors

## Request types

- Bug: use the bug report template
- Enhancement: use the feature request template
- Security: follow the [Security Policy](./SECURITY.md)
